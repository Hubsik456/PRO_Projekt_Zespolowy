Modele Bazy Danych¶

Model bazy danych dla tabeli «Użytkownicy».

Moduł definiuje model SQLAlchemy dla użytkowników aplikacji, integrując się z Flask-Login do zarządzania sesjami.

class Aplikacja.Modele.Użytkownicy.Uzytkownik(**kwargs)[źródło]¶

Model tabeli «Uzytkownicy» w bazie danych.

Reprezentuje użytkownika aplikacji i przechowuje jego dane. Dziedziczy z UserMixin dla kompatybilności z Flask-Login.

Zmienne:

  • id (int) – Unikalny identyfikator użytkownika (klucz główny).

  • login (str) – Login użytkownika, musi być unikalny.

  • haslo (str) – Zahaszowane hasło użytkownika.

  • email (str) – Adres email użytkownika, musi być unikalny.

  • dodano (datetime) – Data i czas utworzenia konta.

  • opis (str) – Opcjonalny opis profilu użytkownika.

  • przedmioty_uzytkownika (list) – Lista przedmiotów posiadanych przez użytkownika.

get_id()[źródło]¶

Zwraca ID użytkownika, wymagane przez Flask-Login.

Zwraca:

ID użytkownika jako string.

Typ zwracany:

str

Model bazy danych dla tabeli «Przedmioty».

Główny moduł modelu kolekcji, definiujący strukturę przedmiotu, który jest centralnym elementem aplikacji.

class Aplikacja.Modele.Kolekcja_Przedmioty.Przedmiot(**kwargs)[źródło]¶

Model tabeli «Przedmioty» w bazie danych.

Reprezentuje pojedynczy przedmiot w kolekcji użytkownika.

Zmienne:

  • id (int) – Unikalny identyfikator przedmiotu (klucz główny).

  • nazwa (str) – Nazwa przedmiotu.

  • opis (str) – Opcjonalny opis przedmiotu.

  • data_dodania (datetime) – Data i czas dodania przedmiotu.

  • data_edycji (datetime) – Data i czas ostatniej edycji przedmiotu.

  • id_wlasciciel (int) – Klucz obcy wskazujący na właściciela przedmiotu.

  • cena_zakupu (Decimal) – Cena, za jaką przedmiot został nabyty.

  • id_cena_zakupu_waluta (int) – Klucz obcy waluty dla ceny zakupu.

  • wartosc_rynkowa (Decimal) – Aktualna wartość rynkowa przedmiotu.

  • id_wartosc_rynkowa_waluta (int) – Klucz obcy waluty dla wartości rynkowej.

  • id_kategoria (int) – Klucz obcy wskazujący na kategorię przedmiotu.

  • czy_prywatne (bool) – Flaga określająca, czy przedmiot jest widoczny publicznie.

Model bazy danych dla tabeli «Kategorie».

Moduł definiuje model SQLAlchemy dla kategorii, do których mogą być przypisywane przedmioty z kolekcji.

class Aplikacja.Modele.Kolekcja_Kategorie.Kategoria(**kwargs)[źródło]¶

Model tabeli «Kategorie» w bazie danych.

Reprezentuje kategorię, którą można przypisać do przedmiotu.

Zmienne:

  • id (int) – Unikalny identyfikator kategorii (klucz główny).

  • nazwa (str) – Nazwa kategorii, musi być unikalna.

  • opis (str) – Opcjonalny, dłuższy opis kategorii.

  • przedmioty (list) – Lista przedmiotów powiązanych z tą kategorią.

Model bazy danych dla tabeli «Waluty».

Moduł definiuje model SQLAlchemy dla walut, które są używane do określania ceny zakupu i wartości rynkowej przedmiotów.

class Aplikacja.Modele.Kolekcja_Waluty.Waluta(**kwargs)[źródło]¶

Model tabeli «Waluty» w bazie danych.

Przechowuje informacje o walutach (np. PLN, USD, EUR).

Zmienne:

  • id (int) – Unikalny identyfikator waluty (klucz główny).

  • skrot (str) – Trzyliterowy skrót waluty (np. „PLN”), musi być unikalny.

  • nazwa (str) – Pełna nazwa waluty (np. „Polski złoty”), musi być unikalna.

Model bazy danych dla tabeli «Notatki».

Moduł definiuje model SQLAlchemy dla notatek, które można dodawać do poszczególnych przedmiotów w kolekcji.

class Aplikacja.Modele.Kolekcja_Notatki.Notatka(**kwargs)[źródło]¶

Model tabeli «Notatki» w bazie danych.

Reprezentuje notatkę powiązaną z konkretnym przedmiotem.

Zmienne:

  • id (int) – Unikalny identyfikator notatki (klucz główny).

  • id_przedmiot (int) – Klucz obcy wskazujący na przedmiot, do którego należy notatka.

  • tytul (str) – Tytuł notatki.

  • opis (str) – Treść notatki.

  • czy_prywatne (bool) – Flaga określająca, czy notatka jest widoczna publicznie.

  • przedmiot (Przedmiot) – Relacja do obiektu nadrzędnego Przedmiot.

Model bazy danych dla tabeli «Zdjecia».

Moduł definiuje model SQLAlchemy dla zdjęć, które można przypisać do przedmiotów w kolekcji.

class Aplikacja.Modele.Kolekcja_Zdjęcia.Zdjecie(**kwargs)[źródło]¶

Model tabeli «Zdjecia» w bazie danych.

Reprezentuje zdjęcie powiązane z konkretnym przedmiotem.

Zmienne:

  • id (int) – Unikalny identyfikator zdjęcia (klucz główny).

  • id_przedmiot (int) – Klucz obcy wskazujący na przedmiot, do którego należy zdjęcie.

  • zdjecie_dane (bytes) – Binarne dane pliku graficznego.

  • tytul (str) – Tytuł zdjęcia.

  • opis (str) – Opcjonalny opis zdjęcia.

  • mimetype (str) – Typ MIME pliku (np. «image/jpeg»).

  • przedmiot (Przedmiot) – Relacja do obiektu nadrzędnego Przedmiot.

Model bazy danych dla tabeli «Pola_Wlasne».

Moduł definiuje model SQLAlchemy dla wartości niestandardowych pól, które użytkownik może dodać do swoich przedmiotów w kolekcji.

class Aplikacja.Modele.Kolekcja_Pola_Własne.PoleWlasne(**kwargs)[źródło]¶

Model tabeli «Pola_Wlasne» w bazie danych.

Przechowuje konkretną wartość dla niestandardowego pola przypisanego do przedmiotu.

Zmienne:

  • id (int) – Unikalny identyfikator pola (klucz główny).

  • id_rodzaj (int) – Klucz obcy wskazujący na rodzaj tego pola.

  • id_przedmiot (int) – Klucz obcy wskazujący na przedmiot, do którego należy to pole.

  • wartosc (str) – Wartość pola, przechowywana jako tekst.

  • przedmiot (Przedmiot) – Relacja do obiektu nadrzędnego Przedmiot.

  • typ_pola_wlasnego (PoleWlasneRodzaj) – Relacja do obiektu definiującego typ tego pola.

Model bazy danych dla tabeli «Pola_Wlasne_Rodzaje».

Moduł definiuje model SQLAlchemy dla rodzajów niestandardowych pól, które użytkownik może tworzyć i przypisywać do przedmiotów.

class Aplikacja.Modele.Kolekcja_Pola_Własne_Rodzaje.PoleWlasneRodzaj(**kwargs)[źródło]¶

Model tabeli «Pola_Wlasne_Rodzaje» w bazie danych.

Definiuje szablon dla niestandardowego pola (np. „Numer seryjny”, „Materiał”).

Zmienne:

  • id (int) – Unikalny identyfikator rodzaju pola (klucz główny).

  • nazwa (str) – Nazwa rodzaju pola (np. „Materiał”), musi być unikalna.

  • typ_danych (str) – Określa typ danych dla tego pola (np. „tekst”, „liczba”).

  • opis (str) – Opcjonalny opis rodzaju pola.

  • powiazane_pola_wlasne (list) – Lista konkretnych wartości pól tego rodzaju.
